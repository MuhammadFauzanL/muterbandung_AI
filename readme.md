## 1. Penjelasan Aplikasi MuterBandung
**MuterBandung** hadir sebagai inovasi platform rekomendasi wisata cerdas berbasis *AI for Smart Tourism Experience* yang dirancang untuk mentransformasi pengalaman liburan di wilayah Bandung Raya. Aplikasi ini memberikan solusi holistik untuk membantu wisatawan menyusun jadwal perjalanan (*itinerary*) secara mudah, cepat, dan sangat terpersonalisasi.

Lebih dari sekadar direktori pariwisata statis, MuterBandung digerakkan oleh **empat pilar kecerdasan buatan utama** yang bekerja secara sinergis:
- **Model Sentimen Analisis:** Memvalidasi kualitas destinasi dan penginapan secara objektif berdasarkan ekstraksi ulasan riil pengunjung.
- **Behavioral AI:** Mempelajari pola interaksi dan kebiasaan pengguna untuk menyajikan kurasi rekomendasi yang dinamis dan beradaptasi dengan selera di setiap kunjungan.
- **Filter Preferensi:** Memahami konteks pencarian dan otomatis mengombinasikannya dengan kalkulasi geospasial (jarak radius) untuk pemfilteran rute dan *budget* yang sangat presisi.
- **Chatbot Kontekstual (Cepot AI):** Sistem *hybrid* yang hadir sebagai asisten virtual interaktif untuk melengkapi informasi logistik pariwisata secara instan.

## 2. Latar Belakang Masalah
Berdasarkan data statistik, jumlah kunjungan wisatawan ke Kota Bandung telah menembus angka 6,5 juta orang hingga triwulan III 2025, dengan jutaan pergerakan mobilitas tercatat di awal tahun 2026. Namun, tingginya antusiasme pariwisata ini tidak diimbangi dengan kemudahan perencanaan perjalanan. Wisatawan masih sering kewalahan dan menghadapi empat kendala utama di lapangan:
1. **Kesulitan Menentukan Destinasi Objektif:** Wisatawan bingung memilih tempat yang benar-benar berkualitas di tengah banyaknya pilihan dan tumpukan ulasan yang membingungkan.
2. **Inefisiensi Penyusunan Logistik:** Pencarian manual seringkali tidak akurat dan memakan waktu lama untuk merangkai rute destinasi dengan penginapan terdekat yang strategis.
3. **Rentan Mengalami *Overbudget*:** Kesulitan dalam mengalkulasi dan mengombinasikan harga tiket destinasi dengan penginapan yang sesuai dengan batasan anggaran spesifik secara otomatis.
4. **Ketiadaan Pemandu Interaktif:** Aplikasi *travel* saat ini dinilai masih kaku, hanya memberikan daftar rekomendasi umum, dan belum memiliki kemampuan untuk melayani tanya-jawab spesifik secara personal.

## 3. Tujuan Proyek
Menjawab latar belakang masalah tersebut, MuterBandung dibangun dengan tujuan utama untuk menyelesaikan kesulitan wisatawan melalui pendekatan arsitektur *Artificial Intelligence* yang cerdas dan solutif:
- **Memastikan Kualitas Secara Objektif:** Menerapkan *Model Sentimen Analisis* agar pengguna tidak perlu membaca ulasan secara manual; sistem otomatis menyeleksi destinasi dan penginapan yang terjamin kepuasannya.
- **Pencarian Cerdas & Kontrol *Budget* Presisi:** Memanfaatkan *Pencarian Semantik* dan perhitungan jarak untuk merangkai kombinasi rute wisata dan akomodasi yang efisien, sekaligus memastikan total biaya tetap aman dari *overbudget*.
- **Menciptakan Aplikasi yang Mengenali Pengguna:** Mengintegrasikan *Behavioral AI* agar antarmuka aplikasi menjadi cerdas, bebas dari pengaturan *filter* yang berulang, dan semakin relevan dengan gaya liburan pengguna seiring waktu.
- **Menghadirkan Pemandu Virtual Proaktif:** Membangun *chatbot hybrid* dengan model bahasa (*LLM*) yang telah dibekali konteks pariwisata lokal Bandung, siap melayani diskusi spesifik guna melengkapi detail yang tidak tertera pada layar rute perjalanan utama.

## 4. Teknologi yang Digunakan
Proyek ini dibangun dengan dua service utama yang saling terhubung:
- **Frontend:** Next.js 16 (App Router), React 19, TypeScript, Tailwind CSS 4, Lucide React, Leaflet/React Leaflet, html2canvas, dan jsPDF.
- **Backend:** FastAPI (Python), Uvicorn, PostgreSQL, SQLAlchemy, Alembic, Pydantic Settings, JWT Auth, dan pytest untuk pengujian.
- **AI Models:** Model analitik (Sentimen & Klasifikasi) yang telah dilatih secara khusus dapat diakses melalui Hugging Face: [muterbandung-ai-models](https://huggingface.co/fauzanlubada/muterbandung-ai-models).
- **AI Recommender Engine:** SentenceTransformer (`firqaaa/indo-sentence-bert-base`) untuk pencarian semantik berbahasa Indonesia, dengan pipeline LLM 2-layer failover:
  - **Layer 1 (Primary):** OpenRouter API — `google/gemma-2-9b-it:free` (gratis, ~1-3 detik warm response).
  - **Layer 2 (Fallback):** Cloudflare Workers AI — `@cf/meta/llama-3.3-70b-instruct-fp8-fast`.
  - **Layer 3 (Deterministic):** Fallback lokal tanpa LLM — menggunakan ranking engine murni.

## 5. Fitur Unggulan
- **Rekomendasi Filter Preferensi:** Secara cerdas menyamakan kata "murah" dengan "gratis" atau "terjangkau", serta mendeteksi nuansa seperti "alam", "romantis", atau "edukasi".
- **Personalisasi & Behavioral AI:** Mempelajari pola interaksi dan kebiasaan pengguna untuk memberikan rekomendasi perjalanan yang dinamis dan relevan di setiap kunjungan.
- **AI Chatbot:** Asisten *virtual* yang memberikan saran spesifik dan alasan logis berdasarkan dataset.
- **Rekomendasi Penginapan Sekitar:** Secara otomatis memetakan dan menyarankan hotel/villa terdekat dari wisata pilihan Anda.

## 6. Arsitektur Aplikasi
Aplikasi ini memecah beban kerjanya ke dalam dua *services* utama agar pengembangan frontend dan backend tetap terpisah:

- **Frontend (Next.js)**
  $\rightarrow$ Menangani antarmuka UI/UX interaktif untuk pengguna, berjalan di `http://localhost:3000`, dan meneruskan request API melalui konfigurasi `API_URL` / `NEXT_PUBLIC_API_URL`.
- **Backend API (FastAPI)**
  $\rightarrow$ Menyediakan endpoint destinasi, penginapan, preferensi, rekomendasi, autentikasi, favorit, event pengguna, dan planner. Backend terhubung ke PostgreSQL dan berjalan di `http://localhost:8000`.

## 8. Cara Instalasi Lokal untuk Development

### Struktur Folder
```text
📦 MuterBandung
 ┣ 📂 backend/                 # Backend API (FastAPI + PostgreSQL)
 ┣ 📂 frontend/                # Aplikasi Antarmuka (Next.js)
 ┣ 📂 ai_workspace/            # Workspace/dataset pendukung
 ┣ 📄 README_DEV.md            # Catatan development tambahan
 ┗ 📄 readme.md                # Dokumentasi utama proyek
```

### Prasyarat
- Git
- Node.js 20+ dan npm
- Python 3.9+ (3.11+ disarankan)
- Docker dan Docker Compose untuk menjalankan PostgreSQL lokal

### 1. Clone Repository
```bash
git clone https://github.com/MuhammadFauzanL/muterbandung_AI.git
cd muterbandung_AI
```

### 2. Setup Backend
```bash
cd backend
cp .env.example .env
docker compose up -d

python -m venv venv
source venv/bin/activate
# Windows: venv\Scripts\activate

pip install -r requirements.txt
alembic upgrade head
uvicorn app.main:app --reload
```

Backend berjalan di `http://localhost:8000`.
Dokumentasi API tersedia di `http://localhost:8000/docs`.

> **Catatan Penting:**
> - Pastikan `DATABASE_URL` di file `.env` mengarah ke port PostgreSQL yang benar (default proyek ini: `5433`).
> - Untuk mengaktifkan Chatbot Cepot AI dengan LLM, tambahkan variabel berikut di `.env`:
>   ```env
>   MUTERBANDUNG_ENABLE_ONLINE_CHAT_LLM=true
>   OPENROUTER_API_KEY=sk-or-v1-xxxxxxxx   # Dapatkan gratis di https://openrouter.ai
>   ```
> - Pada panggilan pertama ke Cepot, sistem akan mengunduh model SentenceTransformer (~100MB) ke cache lokal. Proses ini hanya terjadi sekali.

### 3. Import Data Awal (Opsional)
Jalankan dari folder `backend` setelah migration selesai jika database lokal masih kosong:

```bash
python scripts/import_destinations.py --file data/wisata_populer.csv
python scripts/import_accommodations.py --file data/penginapan_data.csv
python scripts/import_destination_gallery_images.py --file data/wisata_image.csv --apply
```

### 4. Setup Frontend
Buka terminal baru dari root repository:

```bash
cd frontend
cp .env.local.example .env.local
npm install
npm run dev
```

Frontend berjalan di `http://localhost:3000`.

### 5. Alur Menjalankan Aplikasi
1. Pastikan PostgreSQL aktif melalui `docker compose up -d` di folder `backend`.
2. Jalankan backend dengan `uvicorn app.main:app --reload`.
3. Jalankan frontend dengan `npm run dev` di folder `frontend`.
4. Buka `http://localhost:3000` di browser.
